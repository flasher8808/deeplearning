Diskussion

  • Die Trainingsdaten des Modells enthalten viele Bilder, auf denen nicht nur ein Objekt zu sehen ist, sondern mehrere und mit der Bezeichnung keine klare Zuordnung vorgenommen werden kann (Beispiel: rubber_eraser -> zusehen auch Bleistift, Notenbuch, Notenständer)
  • Manche Gegenstände sind in falscher Größe dargestellt (Beispiel: safety_pin)
  • Trainingsdaten enthalten nur ein bestimmtes Set an Objekten / Klassifikationen -> Wird ein Bild mit unbekannten Dingen gezeigt, wird nur bestmöglichste Klasse gefunden was dennoch falsch ist
Bei eindeutigen Bildern, die aus dem Trainingsdatensatz stammen erfolgt meist eine korrekte Klassifikation. Dokumentation

Technisch

Die Web-Anwendung basiert auf einer HTML-Datei, die mit CSS gestylt und JavaScript interaktiv erweitert wurde. Zusätzlich wurden folgende Frameworks verwendet:

  • ml5js: Klassifikation der Eingabebilder auf Basis des vortrainierten neuronalen Netzes
  • Plotly: Darstellung der Klassifikationsergebnisse in einem Diagramm

  1. Entwicklung der HTML-Basis
